The Nine Box Exercise

An experiment with bootstrap rows and containers

This was a really useful exercise for me in figuring out bootstrap rows and containers. I started with a basic design: a full width header and footer accross the top and bottom. In the middle, a block with a main article with three subs below on the left, an a sidebar on the right. All together, when I boxified it, I got nine boxes, so I'm calling it my nine box model, and the task of figuring it out was my nine box exercise..

The Task

Build a page that mimics this one. The header and footer should extend accross 12 columns, the middle should be broken up into a sidebar, a main article, and three sub articles below it. All sections should take their full space, and all margins should nest neatly inside of each other.

How I did it

It seemed like it was going to be pretty easy, I thought I understood how rows and columns worked. Turns out... not so much. I started by boxifying the layout and translating that into an HTML skeleton, and I included a reference to the bootsrap CDN in my header. Then I added some text to see where it showed up and things looked pretty good... then I looked closer. See the sidebar for the things I screwed up and how I fixed them.

Why I did it

I'm learning. I've been taking various front end software courses, and I can do the exercises that they're presenting me with, but there's something to be said for trying to make it up yourself. So, I'm starting to just see what I can do. I hope that publishing the exercises and the results will be useful to others. I'm finding the help I see from google searches to be awesome, but sometimes not quite at my (elementary) level.

Things I learned

I thought, of course, that this was going to be easy... but not so much. Spoiler alert! What follows is a list of things that I had to figure out. Try it yourself before reading this, and then maybe just peak at one thing at a time to see if it helps. The text below tiny. To see it - either change the size in dev tools, or copy and paste it into a text document. If you do it in dev tools, you can do one item at a time.

Label your HTML

Nesting divs are crazy confusing. Label your div start and ends with comments. This side bar, for example, has "container for side bar" after the open div and "close container for side bar" after the closing tag. This especially heled when I started moving things around. My first problem was that I couldn't get this side bar on the side - so I moved it a lot.

Use Borders

Make colored borders, so you can see what's happening and where. I've left them on so you can see. the red border is the container for the whole page, and then I go in rainbow order. Orange for the header and footer, yellow for the mid section, green for the to big boxes in the mid section, blue for the two parts of the main/sub box, and purple for the individual parts of the main/sub box. It took a while to get the borders to show up. I've been messing up on borders a lot lately, so I had to take a detour to figure out what I was doing wrong. Turns out, I was leaving out the border style; ".jredborder {border: 4px red}" doesn't work ".jborder {4px solid red}" does. I created a rainbow of border classes: .jredborder, .jorangeborder, etc. By applying that class to different HTML elements, I was able to see where things were going awry.

Containers vs Rows

This was probably in some of my classes.... but I sure don't remember it. I knew that I had to use a container to hold the whole page... but I didn't know I needed them within the page. Turns out, any place you have a box that needs more than one row (the main/sub box, for example) you need a container first, and then rows inside. This layout has two containers - on for the whole page (red border), and one for the main article and three subs (dark green border). The other mistake I made here, was not having enough rows. For the main article, I tried to use the same div to hold the row and the columns... that didn't work. I didn't really think it would, but I tested it anyway.

How many Columns?

The last thing I had to figure out was why I had extra space to the left of my sidebar. Turns out to be related to the rows and columns probelm above. The container (dark green border) is 9 columns, and the sidebar (light green border) is 3 columns. Inside the container, I had the main article labeld as 9 columns, and the three subs labeled as 3 columns. This was leaving a three column wide space begween the articles and the side bar. Because of the borders, I could see that the container (dark green) was taking up all of the space, but the columns (purple borders) inside weren't. I changed the columns to be 12 columns for the main, and 4 each of the subs, which fixed it. So, it turns out that each container has 12 columns.... I really don't think that was in any of my classes.... If it was, I totally didn't get it!
